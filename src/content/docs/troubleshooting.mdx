# פתרון בעיות נפוצות

מדריך זה נועד לעזור לכם לזהות ולפתור בעיות נפוצות שעלולות להתרחש במהלך ההתקנה, ההרצה או עריכת התוכן של משחק "יציאת מצרים".

## בעיות בהתקנה

### 1. שגיאה: `npm command not found` / `node command not found`

**סיבה אפשרית**: Node.js ו-npm לא הותקנו כראוי או שהנתיב אליהם לא מוגדר במשתני הסביבה של המערכת.

**פתרון**:
1. וודאו שהתקנתם את Node.js (גרסת LTS) מהאתר הרשמי.
2. נסו להתקין מחדש את Node.js.
3. וודאו שהנתיב לתיקיית ההתקנה של Node.js נמצא במשתנה הסביבה `PATH`. (חפשו באינטרנט "add node to path [your operating system]" לקבלת הוראות).
4. הפעילו מחדש את הטרמינל ונסו שוב את הפקודות `node --version` ו-`npm --version`.

### 2. שגיאה: `git command not found`

**סיבה אפשרית**: Git לא הותקן כראוי או שהנתיב אליו לא מוגדר במשתני הסביבה.

**פתרון**:
1. וודאו שהתקנתם את Git מהאתר הרשמי.
2. נסו להתקין מחדש את Git.
3. וודאו שהנתיב לתיקיית ההתקנה של Git נמצא במשתנה הסביבה `PATH`.
4. הפעילו מחדש את הטרמינל ונסו שוב את הפקודה `git --version`.

### 3. שגיאות במהלך `npm install`

**סיבה אפשרית**: בעיות ברשת, קבצים פגומים במטמון של npm, או חוסר התאמה בין גרסאות.

**פתרון**:
1. **בדקו חיבור אינטרנט**: וודאו שיש לכם חיבור אינטרנט יציב.
2. **נקו מטמון npm**: הריצו `npm cache clean --force` בטרמינל (בתיקיית הפרויקט).
3. **מחקו `node_modules` ו-`package-lock.json`**:
   * מחקו את תיקיית `node_modules`.
   * מחקו את הקובץ `package-lock.json`.
   * הריצו שוב `npm install`.
4. **בדקו גרסת Node.js**: וודאו שאתם משתמשים בגרסת Node.js LTS התואמת לפרויקט.

## בעיות בהרצה (`npm run dev`)

### 1. שגיאה: `Port 3000 is already in use`

**סיבה אפשרית**: תהליך אחר במחשב (אולי הרצה קודמת של המשחק שלא נסגרה כראוי) כבר משתמש בפורט 3000.

**פתרון**:
1. **עצרו את התהליך הקיים**:
   * בטרמינל שבו רץ התהליך, לחצו `Ctrl + C`.
   * אם זה לא עוזר, ייתכן שתצטרכו למצוא ולהפסיק את התהליך דרך מנהל המשימות (Windows) או Activity Monitor (macOS).
2. **הריצו בפורט אחר**: נסו להריץ את המשחק בפורט אחר: `npm run dev -- -p 3001` (ואז גשו ל-`http://localhost:3001`).
3. **הפעילו מחדש את המחשב**: במקרים מסוימים, הפעלה מחדש פותרת את הבעיה.

### 2. המשחק לא נטען בדפדפן או מציג שגיאות

**סיבה אפשרית**: שגיאות בקוד, בעיות בתלויות, או בעיות תצורה.

**פתרון**:
1. **בדקו את הטרמינל**: חפשו הודעות שגיאה בטרמינל שבו הרצתם `npm run dev`. הודעות אלו לרוב יצביעו על מקור הבעיה.
2. **בדקו את קונסולת הדפדפן**:
   * פתחו את המשחק בדפדפן.
   * לחצו `F12` (או קליק ימני > Inspect / Inspect Element).
   * עברו ללשונית "Console". חפשו הודעות שגיאה באדום.
3. **וודאו שהתלויות הותקנו**: נסו להריץ שוב `npm install`.
4. **בדקו שינויים אחרונים**: אם הבעיה הופיעה לאחר שביצעתם שינויים, נסו לבטל אותם (אם אתם משתמשים ב-Git, זה קל יותר) ולראות אם הבעיה נפתרת.

## בעיות בעריכת תוכן

### 1. שינויים בקבצי JSON לא משתקפים במשחק

**סיבה אפשרית**:
* הקובץ לא נשמר כראוי.
* שגיאת תחביר בקובץ ה-JSON מונעת ממנו להיטען.
* שרת הפיתוח (`npm run dev`) לא זיהה את השינוי ודורש הפעלה מחדש.

**פתרון**:
1. **וודאו שמירה**: שמרו את הקובץ בעורך הטקסט.
2. **בדקו תקינות JSON**:
   * פתחו את הקובץ בעורך קוד כמו VS Code, הוא לרוב יסמן שגיאות תחביר.
   * השתמשו בכלי אימות JSON מקוון (חפשו "JSON validator online") כדי לבדוק את תקינות הקובץ. תקנו שגיאות (פסיקים חסרים/מיותרים, מרכאות, סוגריים).
3. **הפעילו מחדש את השרת**: עצרו את תהליך `npm run dev` (עם `Ctrl + C`) והריצו אותו מחדש.

### 2. תמונות לא מוצגות במשחק

**סיבה אפשרית**:
* נתיב הקובץ בקובץ ה-JSON שגוי.
* שם הקובץ בקובץ ה-JSON שגוי (כולל אותיות גדולות/קטנות).
* קובץ התמונה לא קיים במיקום המצוין בתיקיית `public`.
* קובץ התמונה פגום.

**פתרון**:
1. **בדקו את הנתיב**: וודאו שהנתיב בקובץ ה-JSON (לדוגמה, `/images/adventures/days_of_honor/prosperity_1.png`) תואם בדיוק למיקום הקובץ בתיקיית `public`. הנתיב צריך להתחיל ב-`/`.
2. **בדקו את שם הקובץ**: וודאו ששם הקובץ ב-JSON זהה לחלוטין לשם הקובץ האמיתי, כולל אותיות גדולות/קטנות (חשוב במיוחד ב-macOS/Linux).
3. **וודאו שהקובץ קיים**: בדקו פיזית שהקובץ נמצא בנתיב המצוין בתיקיית `public`.
4. **נסו לפתוח את התמונה**: נסו לפתוח את קובץ התמונה ישירות כדי לוודא שהוא תקין.
5. **בדקו את קונסולת הדפדפן**: חפשו שגיאות 404 (Not Found) בקונסולה, הן יצביעו על נתיבים שגויים.

### 3. המשחק קורס לאחר עריכת תוכן

**סיבה אפשרית**: לרוב נגרם משגיאת תחביר בקובץ JSON או MDX שעצרתם.

**פתרון**:
1. **בדקו את הטרמינל**: חפשו הודעות שגיאה מפורטות בטרמינל.
2. **בדקו את הקובץ האחרון שערכתם**: חפשו שגיאות תחביר בקובץ ה-JSON או ה-MDX האחרון שנגעתם בו.
3. **בטלו שינויים**: אם אתם משתמשים ב-Git, נסו לחזור ל-commit הקודם ולראות אם הבעיה נפתרת. אם לא, בטלו ידנית את השינויים האחרונים שביצעתם.

## קבלת עזרה נוספת

אם ניסיתם את הפתרונות במדריך זה ועדיין נתקלתם בבעיות, אל תהססו:

* **לעיין במדריכים האחרים**: ייתכן שהתשובה נמצאת במדריך ספציפי יותר.
* **לחפש את הודעת השגיאה**: העתיקו את הודעת השגיאה המדויקת וחפשו אותה באינטרנט.
* **לפנות לצוות הפיתוח**: ספקו תיאור מפורט של הבעיה, הודעות השגיאה שקיבלתם, והשלבים שניסיתם לבצע.
