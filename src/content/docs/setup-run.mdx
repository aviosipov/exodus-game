# התקנה והרצה מקומית

מדריך זה יעזור לך להתקין ולהריץ את משחק "יציאת מצרים" במחשב שלך, גם אם אין לך ניסיון קודם בפיתוח תוכנה.

## דרישות מערכת

לפני שמתחילים, וודא שהמחשב שלך עומד בדרישות הבאות:

1. **מערכת הפעלה**: Windows 10/11, macOS, או Linux
2. **זיכרון RAM**: לפחות 4GB (מומלץ 8GB)
3. **שטח דיסק פנוי**: לפחות 1GB
4. **חיבור אינטרנט**: נדרש להורדת הפרויקט והתקנת התלויות

## התקנת כלים בסיסיים

לפני שנוכל להריץ את המשחק, עלינו להתקין מספר כלים בסיסיים:

### 1. התקנת Node.js ו-npm

Node.js הוא סביבת הריצה שעליה המשחק פועל, ו-npm הוא מנהל החבילות שלו.

1. גש לאתר הרשמי של [Node.js](https://nodejs.org/)
2. הורד את הגרסה ה**LTS (Long Term Support)** המומלצת
3. התקן את Node.js על ידי הרצת הקובץ שהורדת ועקיבה אחר ההוראות

כדי לוודא שההתקנה הצליחה, פתח את חלון הטרמינל (Command Prompt ב-Windows, Terminal ב-macOS/Linux) והקלד:

```bash
node --version
npm --version
```

אם תראה מספרי גרסה (למשל `v20.x.x` ו-`10.x.x`), ההתקנה הצליחה.

### 2. התקנת Git (אופציונלי, אך מומלץ)

Git הוא כלי לניהול גרסאות שיעזור לך להוריד את הפרויקט ולעקוב אחר שינויים.

1. גש לאתר הרשמי של [Git](https://git-scm.com/)
2. הורד את הגרסה המתאימה למערכת ההפעלה שלך
3. התקן את Git על ידי הרצת הקובץ שהורדת ועקיבה אחר ההוראות

כדי לוודא שההתקנה הצליחה, פתח את חלון הטרמינל והקלד:

```bash
git --version
```

אם תראה מספר גרסה (למשל `2.x.x`), ההתקנה הצליחה.

## הורדת הפרויקט

יש שתי דרכים להוריד את הפרויקט:

### אפשרות 1: הורדה באמצעות Git (מומלץ)

1. פתח את חלון הטרמינל
2. נווט לתיקייה שבה תרצה לשמור את הפרויקט (לדוגמה, `Documents` או `Desktop`)
3. הקלד את הפקודה הבאה להורדת הפרויקט:

```bash
git clone https://github.com/[username]/exodus-game.git
```

(החלף את `[username]` בשם המשתמש של בעל המאגר)

4. עבור לתיקיית הפרויקט:

```bash
cd exodus-game
```

### אפשרות 2: הורדה כקובץ ZIP

1. גש לדף הפרויקט ב-GitHub: `https://github.com/[username]/exodus-game`
2. לחץ על כפתור "Code" (ירוק) ובחר "Download ZIP"
3. חלץ את הקובץ שהורדת לתיקייה שתבחר
4. פתח את חלון הטרמינל ונווט לתיקיית הפרויקט שחילצת

## התקנת תלויות הפרויקט

לאחר שהורדת את הפרויקט, עליך להתקין את כל התלויות (חבילות התוכנה) שהפרויקט משתמש בהן:

1. וודא שאתה נמצא בתיקיית הפרויקט בטרמינל
2. הרץ את הפקודה הבאה:

```bash
npm install
```

תהליך זה עשוי להימשך מספר דקות, תלוי במהירות האינטרנט והמחשב שלך.

## הגדרת קובץ סביבה (אם נדרש)

אם הפרויקט דורש הגדרות סביבה מיוחדות:

1. העתק את קובץ `.env.local.sample` לקובץ חדש בשם `.env.local`:

```bash
# ב-Windows
copy .env.local.sample .env.local

# ב-macOS/Linux
cp .env.local.sample .env.local
```

2. ערוך את הקובץ `.env.local` והוסף את המפתחות הנדרשים (אם יש צורך)

## הרצת המשחק בסביבת פיתוח

כעת אתה מוכן להריץ את המשחק במצב פיתוח:

1. וודא שאתה נמצא בתיקיית הפרויקט בטרמינל
2. הרץ את הפקודה הבאה:

```bash
npm run dev
```

3. המתן עד שתראה הודעה שהשרת פועל (בדרך כלל על פורט 3000)
4. פתח את הדפדפן וגש לכתובת: `http://localhost:3000`

כעת המשחק אמור לרוץ בדפדפן שלך!

## בניית המשחק לייצור (אופציונלי)

אם תרצה ליצור גרסת ייצור של המשחק (למשל, לצורך העלאה לשרת):

1. הרץ את הפקודה הבאה:

```bash
npm run build
```

2. לאחר סיום הבנייה, הרץ את הפקודה הבאה להרצת גרסת הייצור:

```bash
npm run start
```

3. פתח את הדפדפן וגש לכתובת: `http://localhost:3000`

## פתרון בעיות נפוצות

### הפקודה `npm` לא מזוהה

אם אתה מקבל שגיאה ש-`npm` אינו מזוהה, וודא שהתקנת את Node.js כראוי. נסה להתקין מחדש או להוסיף את הנתיב ל-Node.js למשתנה הסביבה PATH.

### שגיאות בהתקנת התלויות

אם אתה נתקל בשגיאות במהלך `npm install`, נסה:

1. מחק את תיקיית `node_modules` (אם קיימת)
2. מחק את הקובץ `package-lock.json` (אם קיים)
3. הרץ שוב `npm install`

### השרת לא עולה

אם השרת לא עולה או שאתה מקבל שגיאות בהרצת `npm run dev`:

1. וודא שאין תהליך אחר שכבר משתמש בפורט 3000
2. נסה לסגור ולפתוח מחדש את הטרמינל
3. וודא שהתקנת את כל התלויות כראוי

### בעיות נוספות

אם אתה נתקל בבעיות אחרות, ראה את [מדריך פתרון הבעיות](/docs/troubleshooting) או פנה לצוות הפיתוח לעזרה.

המדריכים הבאים יסבירו כיצד לערוך את תוכן המשחק ולהתאים אותו לצרכים שלך.
