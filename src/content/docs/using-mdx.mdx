# שימוש ב-MDX

מדריך זה מספק הסבר קצר על MDX (Markdown with JSX) וכיצד הוא משמש בפרויקט "יציאת מצרים", בעיקר עבור יצירת דפי התיעוד שאתם קוראים כעת.

## מה זה MDX?

MDX הוא הרחבה של שפת הסימון הפופולרית Markdown. Markdown מאפשרת לכתוב טקסט מעוצב (כמו כותרות, רשימות, הדגשות) בצורה פשוטה וקריאה, ללא צורך ב-HTML מורכב.

MDX לוקח את הפשטות של Markdown ומוסיף לה את היכולת לשלב רכיבי React (JSX) ישירות בתוך הטקסט. זה מאפשר ליצור דפים דינמיים ואינטראקטיביים יותר מאשר דפי Markdown רגילים.

## כיצד MDX משמש בפרויקט זה?

בפרויקט "יציאת מצרים", MDX משמש בעיקר ליצירת דפי התיעוד (הקבצים בתיקייה `src/content/docs`).

השימוש ב-MDX מאפשר לנו:

1. **לכתוב תוכן בקלות**: רוב התיעוד נכתב ב-Markdown פשוט ומוכר.
2. **לעצב את התוכן**: להשתמש בתגי Markdown ליצירת כותרות, רשימות, קישורים, הדגשות וכו'.
3. **להוסיף רכיבים אינטראקטיביים**: לשלב רכיבי React מותאמים אישית בתוך התיעוד (לדוגמה, הצגת קטעי קוד מעוצבים, טבלאות מיוחדות, או אפילו רכיבים אינטראקטיביים).
4. **לנהל מטא-דאטה**: להגדיר כותרת ותיאור לכל דף תיעוד באמצעות "Frontmatter" בתחילת הקובץ (החלק שבין `---`).

## דוגמה לשימוש ב-MDX

```mdx
---
title: דוגמה לשימוש ב-MDX
description: דף זה מדגים כיצד ניתן לשלב Markdown ו-JSX
---

# כותרת ראשית (Markdown)

זוהי פסקה רגילה ב-Markdown. אפשר להשתמש ב**הדגשה** או ב*כתב נטוי*.

## כותרת משנה (Markdown)

*   זוהי רשימה
*   עם מספר פריטים

{/* כאן אפשר לשלב רכיב React */}
<MyCustomComponent prop1="value" />

עוד טקסט ב-Markdown...
```

## עריכת קבצי MDX

כדי לערוך את דפי התיעוד:

1. פתחו את קובץ ה-`.mdx` הרלוונטי בתיקייה `src/content/docs` בעורך טקסט.
2. ערכו את התוכן באמצעות תחביר Markdown רגיל.
3. אם אתם מכירים React, תוכלו גם לשלב רכיבים קיימים או ליצור חדשים (זהו נושא מתקדם יותר).
4. שמרו את הקובץ.

## סיכום

MDX הוא כלי גמיש המשלב את הפשטות של Markdown עם העוצמה של React. בפרויקט זה, הוא משמש בעיקר ליצירת דפי תיעוד עשירים וקריאים. רוב הזמן, תוכלו לערוך את התיעוד באמצעות Markdown פשוט.
