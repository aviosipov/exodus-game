# סביבת פיתוח בסיסית

מדריך זה מיועד למשתמשים שרוצים לערוך תוכן או אפילו קוד במשחק "יציאת מצרים", אך אין להם רקע נרחב בפיתוח תוכנה. נסקור כאן מספר כלים בסיסיים שיקלו עליכם את העבודה.

## 1. עורך קוד (Text Editor)

עורך קוד הוא תוכנה המיועדת לכתיבה ועריכה של קוד וקבצי טקסט, כמו קבצי ה-JSON וה-MDX שבהם משתמש המשחק. עורך קוד טוב מציע תכונות כמו הדגשת תחביר (Syntax Highlighting), השלמה אוטומטית וחיפוש מתקדם, שמקלות מאוד על העבודה.

### המלצה: Visual Studio Code (VS Code)

[Visual Studio Code](https://code.visualstudio.com/) הוא עורך קוד חינמי, פופולרי ועוצמתי מבית מיקרוסופט. הוא זמין ל-Windows, macOS ו-Linux.

**יתרונות עיקריים:**

* **חינמי וקוד פתוח**
* **תמיכה מובנית בשפות רבות**: כולל JavaScript, TypeScript, JSON, Markdown (ולכן גם MDX)
* **הדגשת תחביר מעולה**: מקל על קריאת קבצי JSON ו-MDX
* **טרמינל מובנה**: מאפשר להריץ פקודות (כמו `npm run dev`) ישירות מתוך העורך
* **הרחבות רבות**: ניתן להוסיף תכונות נוספות באמצעות הרחבות (למשל, לבדיקת תקינות JSON)
* **אינטגרציה עם Git**: מקל על ניהול גרסאות (ראו בהמשך)

**כיצד להתקין:**

1. גשו לאתר [Visual Studio Code](https://code.visualstudio.com/)
2. הורידו את הגרסה המתאימה למערכת ההפעלה שלכם
3. התקינו את התוכנה

**כיצד לפתוח את הפרויקט:**

1. פתחו את VS Code
2. בחרו `File` > `Open Folder...` (או `קובץ` > `פתח תיקייה...`)
3. נווטו לתיקייה שבה שמרתם את פרויקט "יציאת מצרים" ובחרו אותה

כעת תוכלו לראות את כל קבצי הפרויקט בחלונית הצדדית ולערוך אותם בקלות.

### אלטרנטיבות

אם אתם מעדיפים עורך פשוט יותר:

* **Notepad++** (Windows): עורך טקסט חינמי ופופולרי ל-Windows
* **Sublime Text**: עורך קוד חזק ומהיר (דורש רישיון לשימוש ממושך)
* **Atom**: עורך קוד חינמי וקוד פתוח (פיתוחו הופסק, אך עדיין שמיש)

## 2. טרמינל (Terminal / Command Line)

הטרמינל (נקרא גם "שורת הפקודה" או "קונסולה") הוא ממשק טקסטואלי המאפשר להריץ פקודות במחשב. תצטרכו להשתמש בו כדי:

* להתקין את תלויות הפרויקט (`npm install`)
* להריץ את המשחק בסביבת פיתוח (`npm run dev`)
* להשתמש ב-Git (אם בחרתם)

### כיצד לפתוח את הטרמינל:

* **Windows**: חפשו "Command Prompt" או "PowerShell" בתפריט ההתחלה
* **macOS**: חפשו "Terminal" ב-Spotlight (זכוכית המגדלת) או בתיקיית Utilities
* **Linux**: בדרך כלל זמין בתפריט הראשי או באמצעות קיצור מקשים (כמו Ctrl+Alt+T)
* **בתוך VS Code**: בחרו `Terminal` > `New Terminal` (או `טרמינל` > `טרמינל חדש`)

### פקודות בסיסיות:

* **`cd [directory_name]`**: Change Directory - כניסה לתיקייה (לדוגמה, `cd exodus-game`)
* **`cd ..`**: יציאה תיקייה אחת למעלה
* **`ls`** (macOS/Linux) / **`dir`** (Windows): List - הצגת רשימת הקבצים והתיקיות במיקום הנוכחי

לאחר שפתחתם את הטרמינל, נווטו לתיקיית הפרויקט באמצעות פקודת `cd` לפני הרצת פקודות כמו `npm install` או `npm run dev`.

## 3. ניהול גרסאות עם Git (אופציונלי, אך מומלץ)

Git הוא כלי לניהול גרסאות. הוא מאפשר לכם:

* **לעקוב אחר שינויים**: לראות בדיוק מה השתנה בכל קובץ
* **לשמור היסטוריה**: ליצור "נקודות שמירה" (commits) של הפרויקט
* **לחזור אחורה**: לשחזר גרסאות קודמות אם משהו השתבש
* **לשתף פעולה**: לעבוד יחד עם מפתחים אחרים על אותו פרויקט (פחות רלוונטי לעריכת תוכן בלבד)

### מדוע זה מומלץ?

גם אם אתם רק עורכים תוכן, Git יכול להיות שימושי מאוד:

* **גיבוי**: כל commit הוא סוג של גיבוי. אם מחקתם בטעות קובץ או עשיתם שינוי שגוי, קל לחזור לגרסה קודמת.
* **מעקב**: קל לראות אילו שינויים עשיתם ומתי.

### שימוש בסיסי ב-Git (דרך VS Code)

VS Code מקל מאוד על שימוש בסיסי ב-Git:

1. **הצגת שינויים**: בחלונית הצדדית, לחצו על אייקון ה-Git (נראה כמו צומת עם שלוש נקודות). כאן תוכלו לראות את כל הקבצים ששונו מאז ה-commit האחרון.
2. **הוספת שינויים (Staging)**: לחצו על סימן הפלוס (+) ליד כל קובץ שתרצו לכלול ב-commit הבא.
3. **יצירת Commit**: כתבו הודעה קצרה המתארת את השינויים שביצעתם (לדוגמה, "עדכון טקסט בהרפתקת ימי הכבוד") בשדה ההודעה למעלה, ולחצו על סימן הווי (✓) כדי ליצור את ה-commit.

זהו! יצרתם נקודת שמירה בהיסטוריה של הפרויקט.

### למידה נוספת

אם אתם מעוניינים ללמוד עוד על Git, ישנם מדריכים רבים באינטרנט. התחילו עם הפקודות הבסיסיות: `git status`, `git add`, `git commit`.

## סיכום

שימוש בכלים הנכונים יכול להפוך את תהליך עריכת התוכן והקוד לחלק ויעיל יותר. אנו ממליצים בחום על:

* **Visual Studio Code** כעורך קוד
* שימוש ב**טרמינל המובנה** ב-VS Code
* שימוש בסיסי ב-**Git** (דרך VS Code) למעקב אחר שינויים וגיבוי

כלים אלו יעזרו לכם לעבוד בצורה מסודרת ובטוחה יותר על פרויקט "יציאת מצרים".
