MDX מאפשר לכתוב מסמכי Markdown ולשלב בתוכם קומפוננטות JSX אינטראקטיביות. מדריך זה מסביר כיצד ליצור דפי תיעוד חדשים באמצעות MDX עבור פרויקט זה.

## מבנה קבצים

כל דפי התיעוד צריכים להימצא בתוך התיקייה `src/content/docs/`. ניתן לארגן דפים בתיקיות משנה לפי הצורך.

## מבנה בסיסי

קובץ MDX מתחיל בדרך כלל עם **frontmatter**, ולאחריו התוכן הראשי שנכתב ב-Markdown.

### Frontmatter

Frontmatter נכתב בפורמט YAML בין קווים משולשים (`---`) בראש הקובץ. הוא מספק מטא-דאטה על הדף.

מאפייני frontmatter מרכזיים המשמשים בפרויקט זה:

-   `title`: (חובה) הכותרת הראשית של הדף, המוצגת באופן בולט.
-   `description`: (אופציונלי) תיאור קצר של תוכן הדף, משמש לעתים קרובות ל-SEO ותצוגות מקדימות.
-   `lang`: (מומלץ לדפים בעברית) הגדרת שפת הדף, למשל `he` לעברית, כדי לסייע בכיווניות (RTL).
-   `sidebar`: (אופציונלי) התאמה אישית של התנהגות סרגל הצד (למשל, `label` לכותרת בתפריט, `order` לסדר).

**דוגמה ל-Frontmatter:**

```yaml
---
title: דף התיעוד המדהים שלי
description: למדו הכל על התכונות המדהימות.
lang: he
sidebar:
  label: תכונות מדהימות
  order: 3
---
```

### תוכן

מתחת ל-frontmatter, כתבו את התוכן שלכם באמצעות תחביר Markdown סטנדרטי.

```markdown
---
title: דף לדוגמה
description: זהו דף MDX לדוגמה.
lang: he
---

# כותרת רמה 1

זהו טקסט Markdown סטנדרטי. ניתן להשתמש ב**הדגשה**, *כתב נטוי*, `קוד`, ו[קישורים](https://example.com).

## כותרת רמה 2

- צרו רשימות
- כמו זו

> ציטוטים נתמכים גם כן.

בלוקים של קוד:
```javascript
function greet() {
  console.log('שלום מ-MDX!');
}
```
```

## שימוש בקומפוננטות

MDX מאפשר לשלב קומפוננטות אינטראקטיביות שנכתבו ב-React ישירות במסמך.

1.  **ייבוא הקומפוננטה:**
    השתמשו בתחביר ייבוא סטנדרטי של JavaScript/TypeScript בראש הקובץ (אחרי ה-frontmatter).

2.  **שימוש בקומפוננטה:**
    שלבו את הקומפוננטה באמצעות תחביר JSX ישירות בתוך ה-Markdown היכן שנדרש.

להלן דוגמאות המציגות את ה*קוד* לייבוא ושימוש בקומפוננטות בתוך MDX:

**ייבוא:**
```jsx
import MyComponent from '../../components/MyComponent'; // ייבוא קומפוננטה מותאמת אישית
// ייתכן שתרצו לייבא קומפוננטות UI מהפרויקט עצמו, למשל:
// import { Button } from '@/components/ui/button';
```

**שימוש:**

```jsx
{/* דוגמה לקומפוננטה מותאמת אישית */}
{/* <MyComponent prop1="value" /> */}

{/* דוגמה לקומפוננטת UI מהפרויקט */}
{/* <Button>לחץ כאן</Button> */}
```

## שיטות עבודה מומלצות

-   שמרו על frontmatter תמציתי ומדויק.
-   השתמשו ב-Markdown סמנטי (רמות כותרת נכונות, רשימות וכו').
-   ארגנו את התוכן באופן הגיוני עם כותרות ברורות.
-   אם אתם יוצרים קומפוננטות מותאמות אישית, ודאו שהן מתועדות היטב וממוקמות בתיקייה מתאימה בפרויקט (למשל, `src/components/`).
